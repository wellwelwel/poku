# `sequential`

Run the tests files sequentially.

## CLI

```bash
# Sequential mode

npx poku --sequential ./test
```

## Config File

<Tabs groupId="poku-config">
  <TabItem default value='js' label='poku.config.js'>

```ts
import { defineConfig } from 'poku';

export default defineConfig({
  sequential: true,
});
```

  </TabItem>
  <TabItem value='json' label='.pokurc.jsonc'>

```json
{
  "$schema": "https://poku.io/schemas/configs.json",
  "sequential": true
}
```

  </TabItem>
</Tabs>

<hr />

:::tip
Learn how to control the number of concurrent test executions with the [`concurrency`](/docs/documentation/poku/options/concurrency) option.
:::

:::info
Sequential execution processes test files one by one, but doesn't guarantee alphabetical.<br />
Want to change that? Let's discuss the best approach and the inclusion of a new feature 🤝
:::
