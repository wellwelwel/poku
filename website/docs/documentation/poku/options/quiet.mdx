# `quiet`

Perform tests with no logs. <br />
This option overwrites all `log` settings by exiting with code and no logs (see below).

<History
  records={[
    {
      version: '2.2.0',
      changes: [
        <>
          Support for <code>-q</code> short flag.
        </>,
      ],
    },
  ]}
/>

## CLI

```bash
npx poku --quiet ./test
```

- Short flag: `-q`.

## Config File

<Tabs groupId="poku-config">
  <TabItem default value='js' label='poku.config.js'>

```ts
import { defineConfig } from 'poku';

export default defineConfig({
  quiet: true,
});
```

  </TabItem>
  <TabItem value='json' label='.pokurc.jsonc'>

```json
{
  "$schema": "https://poku.io/schemas/configs.json",
  "quiet": true
}
```

  </TabItem>
</Tabs>

<hr />

:::note
Since **Poku** doesn't transform files, tests using `beforeEach` or `afterEach` with external consoles can be generated.
:::
