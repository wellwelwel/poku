Compartilhe estado e métodos entre arquivos de teste e processos, habilitando padrões avançados de testes de integração e end-to-end.

## O que são Recursos Compartilhados?

**Recursos Compartilhados** permitem que você defina um recurso com estado uma vez e o acesse a partir de múltiplos arquivos de teste e processos. Cada teste obtém a mesma instância de recurso, permitindo que você:

- Compartilhe conexões de banco de dados, armazenamentos em memória ou clientes de API entre testes.
- Coordene estado e asserções entre testes paralelos ou sequenciais.
- Gerencie a lógica de setup e teardown em um único lugar.

O sistema de recursos lida automaticamente com comunicação entre processos, serialização e limpeza — para que você possa focar em escrever testes.

## Uso Básico

### 1. Defina um Contexto de Recurso

Crie um arquivo de recurso e exporte um objeto de contexto:

```ts
// counter.ts
export const CounterContext = {
  name: 'shared-counter',
  factory: () => ({
    count: 0,
    inc() {
      this.count++;
      return this.count;
    },
    get() {
      return this.count;
    },
  }),
};
```

O contexto possui duas propriedades:

- **`name`**: Um identificador único para o recurso compartilhado
- **`factory`**: Uma função que retorna o objeto de recurso com métodos

### 2. Acesse o Recurso nos Testes

Use a função `shared()` para obter o recurso. Ela lida automaticamente com a limpeza:

```ts
import { test, assert, shared } from 'poku';
import { CounterContext } from './counter';

test('incrementar contador', async () => {
  const counter = await shared(CounterContext);

  const current = await counter.get();
  assert.equal(current, 0);

  await counter.inc();
  assert.equal(await counter.get(), 1);
});
```

### 3. Métodos são Chamadas de Procedimento Remoto

Todos os métodos no seu recurso se tornam RPCs assíncronos que funcionam entre processos. O estado é sincronizado automaticamente:

```ts
test('estado compartilhado entre testes', async () => {
  const counter = await shared(CounterContext);

  // Isto obtém o estado atual do recurso
  const value = await counter.get();
  assert.equal(value, 1); // reflete mudanças de outros testes

  // Isto muta o estado compartilhado
  await counter.inc();
});
```

## Exemplo do Mundo Real

Aqui está um exemplo completo de dois testes compartilhando um contador:

**counter.ts**

```ts
export const CounterContext = {
  name: 'shared-counter',
  factory: () => ({
    count: 0,
    inc() {
      this.count++;
      return this.count;
    },
    get() {
      return this.count;
    },
  }),
};
```

**test-a.test.ts**

```ts
import { assert, test, shared } from 'poku';
import { CounterContext } from './counter';

test('Test A: Incrementar Contador', async () => {
  const counter = await shared(CounterContext);

  const current = await counter.get();
  assert.strictEqual(current, 0, 'Deve iniciar em 0');

  await counter.inc();
  assert.strictEqual(await counter.get(), 1, 'Deve ser 1 após incremento');
});
```

**test-b.test.ts**

```ts
import { assert, test, shared, waitForExpectedResult } from 'poku';
import { CounterContext } from './counter';

test('Test B: Verificar Estado do Contador', async () => {
  const counter = await shared(CounterContext);

  // Se você precisar que o resultado do Test A esteja pronto antes de prosseguir,
  // é recomendado aguardar o estado esperado para evitar instabilidade.
  await waitForExpectedResult(counter.get, 1);

  await counter.inc();
  assert.strictEqual(await counter.get(), 2, 'Deve ser 2 após incremento');
});
```

Quando estes testes são executados, eles compartilham a mesma instância do contador. O Test B vê as mudanças de estado feitas pelo Test A.

## Uso Avançado

### Inicialização Assíncrona

Recursos podem ser inicializados de forma assíncrona:

```ts
import mysql from 'mysql2/promise';

export const DatabaseContext = {
  name: 'shared-db',
  factory: async () => {
    const connection = await mysql.createConnection({
      host: 'localhost',
      user: 'root',
      database: 'test',
    });

    return {
      query: (sql) => connection.query(sql),
      close: () => connection.end(),
    };
  },
  cleanup: async (db) => {
    await db.close();
  },
};
```

### Limpeza de Recurso

Para executar limpeza quando o recurso é destruído, use uma função de limpeza opcional na factory:

```ts
export const CacheContext = {
  name: 'shared-cache',
  factory: () => {
    const cache = new Map();

    const resource = {
      set: (key, value) => cache.set(key, value),
      get: (key) => cache.get(key),
    };

    return resource;
  },
  // Função de limpeza opcional
  cleanup: (resource) => {
    resource.clear();
  },
};
```

## Notas Importantes

### Serialização

Argumentos e valores de retorno são serializados usando serialização JSON quando comunicados entre processos. Tipos complexos como funções, referências circulares ou instâncias de classes personalizadas não são suportados. Mantenha seus métodos de recurso simples com tipos básicos: strings, números, booleanos, arrays e objetos.

### Nomenclatura

Cada recurso deve ter um `name` único. Reutilizar o mesmo nome em contextos diferentes fará com que compartilhem estado, o que pode ser útil para testes de integração, mas pode ser inesperado.

### Carregamento Preguiçoso

Recursos são criados sob demanda na primeira vez que você chama `shared(ResourceContext)`. Se um teste nunca acessa um recurso, ele nunca é criado.
