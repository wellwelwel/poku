Quick Comparisons​

Performance​

Poku is continuously tested to ensure the following expectations for basic usage:

  • ~4x faster than Jest (v29.7.0)
  • ~4x faster than Vitest (v2.1.3)
  • ~2x faster than Mocha (v10.7.3) — even with test file isolation
  • You can see how the tests are run and compared in the benchmark directory.
  • Comparing Poku and native test runners (discussion).

Installation Size​

  • ~230x lighter than Vitest
  • ~130x lighter than Jest
  • ~30x lighter than Mocha + Chai

Poku​

Installation​

npm i -D poku tsx

Creating the test file​

test/index.test.ts

import { assert } from 'poku';

assert.deepStrictEqual('1', 1, 'Number should not be a text');

Running tests​

npx poku

That's it 🎉

tip

For simple tests, Poku doesn't need to use test, describe or it, since the message is already in the assert.

  • Poku's assert is just an abstraction from original assert from Node.js.
  • It means: No new learning is needed 🎉

Adopt a Poku for yourself 🩵

Jest​

Installation​

npm i -D jest @types/jest ts-jest

Configuring TypeScript​

Add in your tsconfig.json

{
  "compilerOptions": {
    "esModuleInterop": true
  }
}

Configuring Jest​

jest.config.js

export default {
  preset: 'ts-jest',
  testEnvironment: 'node',
  testMatch: ['**/test/**/*.test.ts'],
};

Creating the test file​

test/index.test.ts

describe('Type comparison', () => {
  test('Number should not be a text', () => {
    expect('1').toStrictEqual(1);
  });
});

Running tests​

npx jest

Mocha + Chai​

Installation​

npm i -D mocha @types/mocha chai @types/chai ts-node

Configuring ts-node​

ts-loader.js

import { register } from 'node:module';
import { pathToFileURL } from 'node:url';

register('ts-node/esm', pathToFileURL('./'));

Configuring Mocha​

.mocharc.json

{
  "spec": "./test/**/*.test.ts",
  "require": "ts-loader.js"
}

Creating the test file​

test/index.test.ts

import { expect } from 'chai';

describe('Type comparison', () => {
  it('Number should not be a text', () => {
    expect('1').to.deep.equal(1);
  });
});

Running tests​

npx mocha

Vitest​

Installation​

npm i -D vitest ts-node

Configuring Vitest​

vitest.config.ts

import { defineConfig } from 'vitest/config';

export default defineConfig({
  test: {
    include: ['test/**/*.test.ts'],
    globals: true,
    environment: 'node',
  },
});

Creating the test file​

test/index.test.ts

import { describe, it, expect } from 'vitest';

describe('Type comparison', () => {
  it('Number should not be a text', () => {
    expect('1').toStrictEqual(1);
  });
});

Running tests​

npx vitest run

AVA​

Installation​

npm i -D ava tsimp

Configuring Git​

Include in the .gitignore:

/.tsimp

Configuring AVA​

Include in the package.json:

{
  "ava": {
    "files": ["test/**/*.test.ts"],
    "extensions": {
      "ts": "module"
    },
    "nodeArguments": ["--import=tsimp"]
  }
}

Creating the test file​

test/index.test.ts

import test from 'ava';

test('Number should not be a text', (t) => {
  t.deepEqual('1', 1);
});

Running tests​

npx ava
